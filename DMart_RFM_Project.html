Section 02

Data Cleaning with MySQL

After importing the Excel file into MySQL (via CSV export), the following structured cleaning process was applied to ensure data integrity before any analysis.

01

Drop Unnamed Index Column

The first column from Excel export was an unlabelled row index. It was dropped using ALTER TABLE dmart_transactions DROP COLUMN `Unnamed: 0`; to prevent it from interfering with queries.

02

Null Check on All Columns

Ran SELECT COUNT(*) WHERE column IS NULL across all 20 columns. Result: zero nulls in any field — the dataset was confirmed complete. No imputation needed.

03

Duplicate Transaction Check

Verified uniqueness of Transaction_ID using GROUP BY ... HAVING COUNT(*) > 1. All 22,000 transaction IDs were unique — no duplicates found.

04

Discount Column Standardisation

Discount was stored as a string (e.g. "10%"). Created a derived numeric column: CAST(REPLACE(Discount, '%', '') AS DECIMAL(5,2)) as Discount_Pct. Used in all aggregation queries.

05

Date Format Validation

Confirmed all Transaction_Date values follow YYYY-MM-DD format and fall within 2025-01-01 to 2025-12-31. No out-of-range dates detected.

06

Data Type Alignment

Customer_Age and Quantity were confirmed as INT; Total_Amount and Unit_Price as DECIMAL/numeric. No mixed-type columns detected. String fields like Category, Gender, Payment_Mode checked for whitespace/case anomalies — all consistent.

07

Referential Consistency

Verified that Customer_ID format follows CUST#### pattern consistently. Confirmed Store_ID follows STORE## format. No orphaned or malformed IDs found across 22,000 records.

✅
Cleaning Result

The dataset was found to be largely clean at source. The key transformation was converting the Discount percentage string to a numeric field and dropping the unnamed index column. All 22,000 records were retained and used for analysis.

Q5
RFM Scoring — Full Customer Table (with NTILE Window Functions)
ADVANCED
⚠️
RFM Logic Explanation

Recency = days since last purchase (lower = better → higher score). Frequency = total visit count (higher = better). Monetary = total spend (higher = better). Each metric is scored 1–5 using NTILE(5). Reference date = 2026-01-01.

MySQL Query
CTE · NTILE · CASE WHEN
WITH rfm_raw AS ( SELECT Customer_ID, Customer_Name, DATEDIFF('2026-01-01', MAX(Transaction_Date)) AS Recency, COUNT(Transaction_ID) AS Frequency, ROUND(SUM(Total_Amount), 2) AS Monetary FROM dmart_transactions GROUP BY Customer_ID, Customer_Name ), rfm_scored AS ( SELECT *, NTILE(5) OVER (ORDER BY Recency ASC) AS R_Score, NTILE(5) OVER (ORDER BY Frequency DESC) AS F_Score, NTILE(5) OVER (ORDER BY Monetary DESC) AS M_Score FROM rfm_raw ) SELECT *, (R_Score + F_Score + M_Score) AS RFM_Total, CASE WHEN (R_Score + F_Score + M_Score) >= 13 THEN 'Champions' WHEN (R_Score + F_Score + M_Score) >= 10 THEN 'Loyal Customers' WHEN (R_Score + F_Score + M_Score) >= 7 THEN 'Potential Loyalists' WHEN (R_Score + F_Score + M_Score) >= 5 THEN 'At Risk' ELSE 'Lost' END AS Customer_Segment FROM rfm_scored ORDER BY RFM_Total DESC;

Final Table Creation Query
CREATE TABLE AS · JOIN
CREATE TABLE dmart_rfm_final AS WITH rfm_raw AS ( SELECT t.Customer_ID, t.Customer_Name, DATEDIFF('2026-01-01', MAX(t.Transaction_Date)) AS Recency, COUNT(t.Transaction_ID) AS Frequency, ROUND(SUM(t.Total_Amount), 2) AS Monetary, MAX(t.Area_Name) AS Area_Name, MAX(t.Gender) AS Gender FROM dmart_transactions t GROUP BY t.Customer_ID, t.Customer_Name ), rfm_scored AS ( SELECT *, NTILE(5) OVER(ORDER BY Recency ASC) R_Score, NTILE(5) OVER(ORDER BY Frequency DESC) F_Score, NTILE(5) OVER(ORDER BY Monetary DESC) M_Score FROM rfm_raw ) SELECT Customer_ID, Customer_Name, Recency, Frequency, Monetary, R_Score, F_Score, M_Score, (R_Score + F_Score + M_Score) AS RFM_Total, CASE WHEN (R_Score+F_Score+M_Score) >= 13 THEN 'Champions' WHEN (R_Score+F_Score+M_Score) >= 10 THEN 'Loyal Customers' WHEN (R_Score+F_Score+M_Score) >= 7 THEN 'Potential Loyalists' WHEN (R_Score+F_Score+M_Score) >= 5 THEN 'At Risk' ELSE 'Lost' END AS Customer_Segment, Area_Name, Gender FROM rfm_scored;
